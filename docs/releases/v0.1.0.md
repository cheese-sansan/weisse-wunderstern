# NoteForge v0.1.0

Initial public prototype release.

## Highlights

- CLI topic/file analysis pipeline.
- Zero-dependency TUI for local interactive use.
- FastAPI service with submit/status/result endpoints.
- Docker Compose deployment with optional document parsing extras.
- Mock mode that runs without API keys.
- Optional OpenAI-compatible LLM mode.
- Public Python integration surface through `core.run_job`.
- Privacy audit script, unit tests, smoke tests, API tests, and CI workflow.

## Security And Privacy

- `.env`, generated outputs, local credentials, and private samples are ignored by default.
- Job IDs and upload filenames are validated.
- LLM HTTP error logging avoids recording upstream response bodies.
- Release validation includes current-file and Git-history privacy audits.

## Known Limitations

- T2 literature candidates are simulated unless a future real provider is added.
- The API is suitable for local or trusted deployment, not hardened multi-tenant hosting.
- Heavy vector databases, GraphRAG, and complex frontends are intentionally out of scope for this first release.

## Recommended Validation

python check_all.py
