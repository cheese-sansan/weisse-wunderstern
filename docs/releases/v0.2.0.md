# NoteForge v0.2.0

NoteForge v0.2 moves the project from a simulated literature prototype to an evidence-aware research pipeline.

## Highlights

- Real Crossref metadata retrieval is now the default and requires no API key.
- Mock and LLM-generated records require explicit selection and remain visibly simulated.
- Every literature record includes provider, source type, retrieval time, DOI/URL fields, and a stable source ID.
- T3 claims carry evidence references; T5 technical cases and T6 policy assessments use structured, source-gated outputs.
- `report.md` includes the active data mode, sources, technical cases, policy assessment, warnings, and trust boundaries.
- CLI, TUI, FastAPI, Docker, and SDK calls share the same Provider selection behavior.

## Failure Semantics

External retrieval timeouts and API errors return an empty evidence set with warnings. Jobs continue to a clearly marked evidence-insufficient report and never silently mix in Mock literature.

Policy facts are emitted only when a source explicitly supports them. Missing policy evidence produces an empty assessment and a request for an authoritative source.

## Compatibility

- `report_framework.md` remains an identical compatibility copy of `report.md`.
- Existing API result fields remain unchanged; v0.2 adds provider status, sources, cases, policy assessment, and warnings.
- Legacy custom Provider records remain readable during the v0.2 migration window.

## Validation

The release is covered by unit tests for Crossref normalization and failure handling, provenance, T5/T6 contracts, resume behavior, CLI/TUI/API smoke tests, privacy auditing, and Docker validation.
