# NoteForge v0.3.0

NoteForge v0.3.0 establishes an installable, strongly typed engineering base. It is a breaking Python/CLI release and a compatible HTTP API v1 release.

Highlights:

- `pip install .` and the unified `noteforge` command on Python 3.10–3.13.
- Standard-library dataclass SDK with `AnalysisRequest`, `JobResult`, typed literature/provider contracts, enums, and structured errors.
- Semantic schema-v3 context and strict Job/Stage transitions.
- Automatic, transactional v0.2 job migration with backups, rollback, report preservation, and explicit bulk migration.
- Wheel-first CI with Ruff, Pyright, pytest coverage ≥80%, OpenAPI compatibility, privacy, API smoke, and Docker gates.

Upgrade requirements:

- Replace `main.py`, `main_tui.py`, and `main_api.py` usage with `noteforge run`, `noteforge tui`, and `noteforge api`.
- Replace imports from `core`, `tasks`, and `utils` with the `noteforge` package.
- Replace requirements files with `.[api]`, `.[documents]`, or `.[dev]` extras.

Citation numbering, evidence grading, report templates, JSON/HTML export, and new providers remain planned for v0.4.
